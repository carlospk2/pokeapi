## Project Overview

Kotlin Multiplatform (KMP) + Compose Multiplatform (CMP) project targeting **Android and iOS**.

- **Package:** `com.example.kmm_jcmm_test`
- **Min SDK:** 26 | **Target/Compile SDK:** 36 | **iOS min:** 16
- **Kotlin:** 2.3.10 | **AGP:** 9.1.0 | **Gradle:** 9.4.0 | **CMP:** 1.10.2 | **JVM toolchain:** 11

## Common Commands

```bash
# Build Android APK
./gradlew :composeApp:assembleDebug
./gradlew :composeApp:assembleRelease

# Build shared module (Android)
./gradlew :shared:compileDebugKotlinAndroid

# Unit tests
./gradlew :shared:testDebugUnitTest
./gradlew :composeApp:testDebugUnitTest

# Install on device
./gradlew :composeApp:installDebug

# Full clean build
./gradlew clean :composeApp:assembleDebug
```

## Module Architecture

```
composeApp/   — Compose Multiplatform UI layer (Android app + iOS framework)
shared/       — KMP business logic: database, networking, DI
iosApp/       — Swift iOS entry point (wraps the ComposeApp framework)
```

**`composeApp`** applies `kotlin.multiplatform` + `com.android.application` + `org.jetbrains.compose`. Contains all Compose UI code. Produces an Android APK and an iOS `ComposeApp.framework`.

**`shared`** applies `kotlin.multiplatform` + `com.android.library` + `androidx.room` + `ksp` + `kotlin.serialization`. Business logic only — no UI. Produces the `Shared.framework` consumed by iOS.

## Key Layers in `shared`

| Package | Contents |
|---|---|
| `shared.database` | Room: `AppDatabase`, `NoteEntity`, `NoteDao`. Platform builders in `androidMain`/`iosMain` |
| `shared.network` | `createHttpClient()` — Ktor client factory (engine injected per platform) |
| `shared.di` | `sharedModule` — Koin module wiring `HttpClient` singleton |

## expect/actual Pattern

- `Platform.kt` — `expect fun getPlatform(): Platform` resolved per target
- `AppDatabaseConstructor` — `expect object` whose `actual` is **generated by KSP** (do not write it manually)
- `colorScheme()` in theme — `expect fun colorScheme(darkTheme: Boolean): ColorScheme`; Android actual uses Material You dynamic color, iOS uses static

## Dependency Management

All versions and library coordinates live in **`gradle/libs.versions.toml`** — never declare inline. Key quirks:

- **CMP 1.10.x**: `compose.*` shorthand aliases are deprecated at error level. Always use explicit catalog entries (`libs.compose.runtime`, `libs.compose.material3`, etc.)
- **`compose-material3`** has independent versioning from the CMP plugin: plugin = `1.10.2`, material3 artifact = `1.9.0`
- **Room KSP**: must be declared per-target in the module-level `dependencies {}` block (not inside `kotlin {}` sourceSets): `add("kspAndroid", ...)`, `add("kspIosX64", ...)`, etc.

## AGP 9 + KMP Compatibility Flags (gradle.properties)

```properties
android.builtInKotlin=false   # prevents conflict between AGP's built-in Kotlin and KMP plugin
android.newDsl=false          # required for com.android.library + kotlin.multiplatform coexistence
```

These flags are **required**. Removing them breaks the build.

## iOS-specific Notes

- iOS frameworks are **static** (`isStatic = true`) in both `shared` and `composeApp`
- `shared` iOS targets require `linkerOpts.add("-lsqlite3")` for Room's `BundledSQLiteDriver`
- Ktor uses `ktor-client-darwin` engine on iOS (in `iosMain`)
- Room database path on iOS uses `NSDocumentDirectory` via `NSFileManager`

## Commit Convention

All commits must follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/):

```
<type>[optional scope]: <description>
```

Common types: `feat`, `fix`, `chore`, `refactor`, `docs`, `test`. Example: `feat(database): add user entity`.

## Adding New Dependencies

1. Add version to `[versions]` in `libs.versions.toml`
2. Add library entry to `[libraries]`
3. Add to the correct sourceSet in the target module's `build.gradle.kts`
4. If it requires a Gradle plugin, also add to `[plugins]` and root `build.gradle.kts` with `apply false`
